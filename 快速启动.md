# 快速启动指南

## 🚀 当前情况

根据你的输出，**后端服务已经成功启动**（端口5000），但是**前端服务没有启动**，因为脚本检测不到Node.js。

## ✅ 解决方案

### 方法1：手动启动前端（推荐）

**在新的命令行窗口中运行：**

1. 打开新的CMD或PowerShell窗口
2. 进入项目目录：
   ```cmd
   cd /d "d:\MetaBCI\emo-system - 副本\emo-system"
   ```
3. 启动前端服务：
   ```cmd
   npm run dev
   ```

### 方法2：使用批处理脚本

双击运行 `启动前端服务.bat`

### 方法3：检查Node.js环境

**问题可能是：**
- Node.js已安装但不在PATH中
- 需要重启命令行窗口

**解决方法：**

1. **检查Node.js是否安装：**
   ```cmd
   node --version
   npm --version
   ```

2. **如果命令找不到，尝试：**
   - 重启命令行窗口
   - 检查Node.js安装路径是否在PATH中
   - 重新安装Node.js

3. **临时解决方案（如果npm不在PATH中）：**
   ```cmd
   "C:\Program Files\nodejs\npm.cmd" run dev
   ```
   或找到你的Node.js安装路径，使用完整路径运行


## 🌐 访问地址

启动前端后，访问：
- **前端界面**：http://localhost:5173
- **后端API**：http://localhost:5000（已运行）

## 🔍 诊断步骤

如果仍然无法启动前端：

1. **检查npm是否可用：**
   ```cmd
   npm --version
   ```

2. **检查Node.js安装：**
   ```cmd
   node --version
   ```

3. **检查项目依赖：**
   ```cmd
   dir node_modules
   ```
   如果不存在，运行：`npm install`

4. **查看详细错误：**
   运行 `npm run dev` 查看具体错误信息

## 💡 提示

- 确保**两个服务都在运行**（后端5000端口 + 前端5173端口）
- 如果npm命令找不到，可能需要重启命令行窗口
- 前端服务启动后，浏览器会自动打开或显示访问地址
