## 一、项目整体成效

### 1. 项目技术路径

本项目 Fork 自 [MetaBCI 官方仓库](https://github.com/TBC-TJU/MetaBCI)，依托 brainstim / brainflow / brainda 三子平台，构建 Web 端四步闭环：

```
brainstim（范式 + Marker）
    ↓
brainflow（采集 / 在线 Worker）
    ↓
brainda（loadmat / 数据集 / LOSO / 评测）
    ↓
应用层模型训练与在线推理
    ↓
Vue 前端可视化（训练曲线 / t-SNE / 脑地形图 / 三分类概率）
```

**各模块职责**

| 阶段 | 子平台 | 关键实现 |
|------|--------|----------|
| 刺激呈现 | brainstim | `LsLPort.setData()` 发送 marker；新增 `SEEDPassiveEmotion` 被动情绪范式（PsychoPy） |
| 数据获取 | brainflow | `ProcessWorker` 封装 `EmotionFeedbackWorker`；`Marker.get_epoch()` 截取 1.5 s epoch；Neuroscan 40 导 LSL 预留 + `neuroscan_sim` 模拟源 |
| 信号处理 | brainda | `loadmat()` 读 SEED `.mat` → 4 导 1.5 s DE 特征；`SEEDEmotionDataset` 统一 Raw 结构；`EnhancedLeaveOneGroupOut` LOSO 划分；`Performance.evaluate()` 输出 Acc/bAcc |
| 应用集成 | 综合 | 五模型特征学习（CADD_DCCNN / CCA_RMPG / DGConformer / EEGMatch / ATGRNet）；Flask REST + Vue 大屏 |

**一键启动**：项目根目录执行 `python start.py`（后端 5001 + 前端 5173）。

### 2. 项目整体效果

基于 SEED **15 被试**、**4 导联**（AF3 / AF4 / F3 / F4）、**1.5 s** 非重叠片段、**5 频段**差分熵（DE）特征，实现积极 / 中性 / 消极三分类。验证方式为 **LOSO**（留一被试，`EnhancedLeaveOneGroupOut`）。

**典型 LOSO 最高验证准确率（debug 模式，held-out subject 0）**

| 模型 | 最高验证准确率 | 训练日志（本地示例，未上传 GitHub） |
|------|----------------|-------------------------------------|
| DGConformer | **85.0%** | `backend/seed/20260704_164243/log.log` |
| ATGRNet | **84.4%** | `backend/seed/20260704_172940/log.log` |
| CADD_DCCNN | **80.0%** | `backend/seed/20260704_205626/log.log` |

#### 2.1 运行环境

| 项目 | 要求 |

#### 2.2 环境安装

```bash
# 1. 克隆仓库
git clone git@github.com:affective-TJNU/emo-system.git
cd emo-system

# 2. 激活 Python 环境
conda activate gkan   # 或自建 Python 3.8+ 环境

# 3. 安装 MetaBCI（brainda + brainflow + brainstim）
cd MetaBCI-master
pip install -e ".[brainda,brainflow,brainstim]"
cd ..

# 4. 安装前端依赖
npm install

# 5. 验证 MetaBCI 三模块可导入
python -c "import metabci.brainda, metabci.brainstim, metabci.brainflow; print('MetaBCI OK')"
```

**关键依赖**：torch、mne、scikit-learn、psychopy、pylsl、flask、flask-cors、numpy、scipy、pandas

**可选环境变量**

| 变量 | 含义 | 默认值 |

| `SEED_EMOTION_ROOT` | SEED 数据根目录 | `backend/seed` |

#### 2.3 数据准备

| 数据 | 路径 | 是否随仓库提供 | 说明 |
|------|------|----------------|------|
| DE 特征张量 | `backend/seed/de_comp_4ch_1p5s/` | **是** | 含 `data.pt`、`label.pt`、`metadata.json`；可直接用于训练 |

1. 确认 `backend/seed/de_comp_4ch_1p5s/data.pt` 存在。
2. 直接启动系统，跳过 raw 重建，进入 Step2 特征学习。

  -H "Content-Type: application/json" \

curl -s http://localhost:5001/api/metabci/brainda/build-status | python3 -m json.tool

#### 2.8 常见问题

| 现象 | 原因 | 处理办法 |
|------|------|----------|
| Step2 长时间停在 88% | 前端等待后端时为**占位进度**，非真实训练进度 | 耐心等待；若 `de_comp_4ch_1p5s` 已存在，训练本身约 1 min |
| 首次训练 log 仅 matplotlib 几行后长时间无输出 | 正在从 `Preprocessed_EEG` 构建 DE（约 8–10 min） | 使用仓库内已有 `de_comp_4ch_1p5s`，或提前完成 Step1 |
| MetaBCI 导入失败 | 未 editable 安装 | `cd MetaBCI-master && pip install -e ".[brainda,brainflow,brainstim]"` |
| 训练报错 / 准确率为 0 | 缺少 DE 特征 | 确认 `backend/seed/de_comp_4ch_1p5s/data.pt` 存在 |
| 在线推理无输出 | 未训练或无 checkpoint | 先完成 Step2，或指定已有模型名 |
| brainstim GUI 无法弹出 | SSH 无 DISPLAY | 使用 `--dry-run`；本地需 `export DISPLAY=:0` |
| 端口占用 | 5001 已被占用 | `export FLASK_PORT=5002` 后重启 |
