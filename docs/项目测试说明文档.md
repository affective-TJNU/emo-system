## 一、项目整体成效

### 1. 项目技术路径

依托 MetaBCI 开源平台架构，本项目构建「数据采集与预处理 → 特征学习 → 在线情绪推理 → 可视化分析」四步闭环，支持 SEED 公开数据集与 Neuroscan 自采双模式。

① **刺激呈现（brainstim）**：调用 `LsLPort.setData` 发送基线/情绪块/EEG 同步/在线反馈 marker（#1~#4、#21~#23）；新增 `SEEDPassiveEmotion` 被动情绪范式（PsychoPy 图片刺激），与 Web 大屏联动。

② **数据获取（brainflow）**：基于 `ProcessWorker` 封装 `EmotionFeedbackWorker` 实现在线推理；通过 `Marker.get_epoch` 截取 1.5s epoch；新增 Neuroscan 40 导 LSL 接入与 `neuroscan_sim` 模拟源，映射至 AF3/AF4/F3/F4 四导联。

③ **信号处理（brainda）**：使用 `loadmat` 读取 SEED 原始 `.mat` 并构建 4 导联 1.5s DE 特征；经 `SEEDEmotionDataset(BaseDataset)` 封装为 MetaBCI 统一 Raw 结构；用 `EnhancedLeaveOneGroupOut` 完成 LOSO 划分；`Performance.evaluate` 输出 Acc/bAcc。

④ **应用集成**：在 MetaBCI 数据边界之上，集成 ATGRNet、DGConformer、EEGMatch 等情绪分类模型，经 brainflow 在线输出三分类概率，Web 端展示训练曲线、t-SNE 与脑地形图。

**技术路径示意**

```
brainstim (范式+Marker) → brainflow (采集/在线Worker) → brainda (loadmat/数据集/LOSO) → 应用模型训练与推理 → Web 可视化
```

**一键启动**：项目根目录 `python start.py`（后端 5001 + 前端 5173）

### 2. 项目整体效果

本项目基于 SEED 15 被试、4 导联（AF3/AF4/F3/F4）、1.5s 非重叠片段与 5 频段差分熵（DE）特征，实现积极 / 中性 / 消极三分类情绪识别。采用 LOSO（留一被试）交叉验证，典型结果如下：

### 2. 代码测试说明

> 本项目为 **Web 全栈系统**（Vue 前端 + Flask 后端 + MetaBCI 三模块）。主办方可按下列步骤完成环境配置、数据准备、系统启动与功能验证。

#### 2.1 运行环境

| 项目 | 要求 |
|------|------|
| 操作系统 | Linux（推荐 Ubuntu 20.04+）；brainstim GUI Demo 需本地桌面 `DISPLAY` |
| Python | 3.8+（实测 conda 环境 `py312`） |
| Node.js | 18+ |
| GPU | 可选（训练加速；CPU 亦可完成演示） |
| 端口 | 后端默认 `5001`，前端默认 `5173`（可通过环境变量 `FLASK_PORT` 修改后端端口） |

#### 2.2 环境安装

```bash
# 1. 进入项目根目录
cd emo-system

# 2. 激活 Python 环境（推荐）
conda activate py312

# 3. 安装 MetaBCI（含 brainda / brainflow / brainstim）
cd MetaBCI-master
pip install -e .[brainda,brainflow,brainstim]
cd ..

# 4. 安装前端依赖
npm install

# 5. 验证 MetaBCI 三模块可导入
python -c "import metabci.brainda; import metabci.brainstim; import metabci.brainflow; print('MetaBCI OK')"
```

**关键 Python 依赖**：mne、torch、scikit-learn、psychopy、pylsl、flask、flask-cors、numpy、scipy、pandas

**可选环境变量**（数据路径非默认位置时设置）：

| 变量 | 含义 |
|------|------|
| `SEED_EMOTION_ROOT` | SEED 数据根目录，默认 `backend/seed` |
| `SEED_EMOTION_DATA_DIR` | 原始 `.mat` 目录，默认 `backend/seed/Preprocessed_EEG` |
| `FLASK_PORT` | 后端端口，默认 `5001` |

#### 2.3 数据准备

| 数据 | 路径 | 是否必需 | 说明 |

```bash
# 方式 A：Web Step1「数据处理」触发（推荐，首次运行会自动从 raw 构建，约需数分钟）
```

#### 2.8 常见问题

| 现象 | 处理办法 |
|------|----------|
| MetaBCI 导入失败 | 确认已执行 `pip install -e .[brainda,brainflow,brainstim]` |
| 训练准确率为 0 或报错 | 确认 `backend/seed/Preprocessed_EEG/` 存在；或先执行 Step1 构建 DE |
| 在线推理无输出 | 先完成 Step2 训练，或选择已有 checkpoint 的模型名 |
| brainstim GUI 无法弹出 | SSH 远程环境仅可 `--dry-run`；本地需 `export DISPLAY=:0` |
| 端口被占用 | 设置 `export FLASK_PORT=5002` 后重启 |

### B. 常见问题

| 问题 | 处理 |
|------|------|
| brainstim verify 失败 | 使用 `conda activate gkan`；确保 pylsl、psychopy、serial 已安装 |
| brainstim GUI 无法弹出 | 需本地桌面 `export DISPLAY=:0`；SSH 远程仅可 `--dry-run` |
| 训练准确率为 0 | 确认 `backend/seed/de_comp_4ch_1p5s/data.pt` 存在 |
| 在线推理无输出 | 先完成 Step2 训练生成 checkpoint，或指定已有 `model_name` |
